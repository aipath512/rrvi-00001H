# Frequently Asked Questions (FAQ)

## EU AI Act Compliance Management System™

Repository: RRVI™

Version: 1.0.2

---

# 1. What is this repository?

This repository contains an engineering implementation of an EU AI Act Compliance Management System for organizations using Artificial Intelligence.

---

# 2. Who should use it?

- SMEs
- Enterprises
- Public organizations
- AI Deployers
- Compliance professionals
- Internal auditors

---

# 3. Does this repository guarantee legal compliance?

No.

It provides a structured Compliance Management System that helps organizations document and demonstrate compliance.

---

# 4. How many modules are included?

Ten modules.

---

# 5. How many controlled documents are included?

Seventy controlled documents.

---

# 6. Why does every module contain seven documents?

Each module follows the same engineering lifecycle:

Policy

↓

Procedure

↓

Work Instruction

↓

Checklist

↓

Register

↓

Evidence

↓

Declaration

---

# 7. Which module should be completed first?

Module 01 — AI System Inventory.

---

# 8. Can modules be completed independently?

No.

Modules are interconnected through traceability.

---

# 9. How often should the system be reviewed?

At least annually and after significant AI, organizational or regulatory changes.

---

# 10. What is objective evidence?

Objective evidence consists of verifiable records demonstrating that compliance activities have actually been performed.

---

# 11. What is a Register?

A Register records operational information required to demonstrate compliance.

---

# 12. What is a Declaration?

A Declaration is management's formal confirmation that compliance activities have been implemented.

---

# 13. What happens during an inspection?

Inspectors review:

- repository structure;
- modules;
- registers;
- evidence;
- declarations;
- traceability.

---

# 14. How are new versions published?

Each release includes:

- Git tag;
- GitHub Release;
- CHANGELOG update;
- Zenodo DOI (when enabled).
