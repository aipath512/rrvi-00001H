# ADMINISTRATION GUIDE

## EU AI Act Compliance Management System™

Repository: RRVI™

Version: 1.0.2

---

# 1. Purpose

This guide describes how administrators maintain the EU AI Act Compliance Management System throughout its operational lifecycle.

---

# 2. Administrator Responsibilities

The Administrator shall:

- maintain repository integrity;
- manage document versions;
- coordinate document reviews;
- manage user access;
- maintain traceability;
- prepare the system for audits and inspections.

---

# 3. Repository Administration

Verify:

- repository structure;
- document naming;
- folder organization;
- version consistency.

Do not rename controlled documents.

---

# 4. Document Administration

Every controlled document shall have:

- unique identifier;
- version;
- approval;
- revision history;
- owner.

---

# 5. Version Management

For every release:

- update CHANGELOG;
- update README;
- create GitHub Release;
- publish Zenodo DOI if applicable.

---

# 6. User Administration

Assign responsibilities for:

- AI Governance Owner;
- AI System Owner;
- Compliance Owner;
- Internal Auditor;
- Repository Administrator.

---

# 7. Backup

Ensure:

- GitHub repository backup;
- document integrity;
- release preservation;
- DOI preservation.

---

# 8. Periodic Reviews

Review:

- annually;
- after regulatory changes;
- after major repository updates.

---

# 9. Inspection Preparation

Before inspection verify:

- all modules complete;
- registers current;
- evidence available;
- declarations approved;
- traceability maintained.

---

# 10. Continuous Maintenance

Maintain:

- repository;
- documentation;
- releases;
- audit records;
- compliance records.
