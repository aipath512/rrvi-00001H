# CONTRIBUTING

## EU AI Act Compliance Management System™

Repository: RRVI™

Version: 1.0.2

---

# 1. Purpose

This document defines the rules for contributing to the EU AI Act Compliance Management System repository.

The objective is to preserve engineering quality, document consistency and complete traceability.

---

# 2. Who Can Contribute

Contributions are welcome from:

- Engineers
- Compliance Specialists
- Internal Auditors
- AI Governance Professionals
- Researchers
- Organizations using the repository

---

# 3. Contribution Principles

Every contribution shall:

- improve the repository;
- preserve traceability;
- maintain document quality;
- respect version control.

---

# 4. Before Contributing

Verify that:

- the issue has not already been solved;
- the proposed change is technically justified;
- document references remain valid.

---

# 5. Allowed Contributions

Examples include:

- document improvements;
- typo corrections;
- regulatory updates;
- engineering improvements;
- additional evidence templates;
- improved checklists.

---

# 6. Not Allowed

Do not:

- rename controlled document IDs;
- change repository structure;
- remove traceability;
- delete controlled documents without justification.

---

# 7. Pull Requests

Every Pull Request should include:

- purpose;
- affected documents;
- impact assessment;
- version update if applicable.

---

# 8. Quality Review

Before approval verify:

- Markdown renders correctly;
- references are valid;
- naming conventions are preserved;
- traceability remains intact.

---

# 9. Versioning

Significant contributions shall:

- update CHANGELOG;
- be included in the next GitHub Release;
- receive a new repository version.
